API

  • cookie-parser used to read cookies
  • cookie-session used to set session in cookie
  • cors used to allow cors being set between requests
  • express used as middleware for log in sessions
  • express-session used as middleware for log in sessions
  • express-validator used as middleware for validating requests
  • gridfs-stream used for uploading files to mongodb
  • mocha used to run tests
  • mongoose used at backend to communicate with mongodb
  • mongoose-gridfs used at backend to communicate with mongodb for files
  • multer used to be able to stream the file data being upload to be stored accordingly
  • multer-gridfs-storage used as middleware for gridfs to communicate with multer and vice versa
  • nodemon used to run application locally for testing
  • passport used as middleware to authenticate session
  • passport-google-oauth used as middleware to authenticate google accounts
  • passport-local used as middleware to authenticate local accounts
  • supertest used for creating tests to ensure application runs smoothly
  • waveform-playlist used for front end to edit music tracks
  • wavesurfer used to display audio as waveforms
